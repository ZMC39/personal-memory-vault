# 发布与隐私规则

公开版本永远从私库生成一份新的净化副本，不直接压缩工作目录，也不修改或删除私库中的原始材料。

从公开副本中排除私人文件只是“副本不携带”，不是删除私库材料，也不构成永久删除授权。类别调整同样只能通过迁移与归档完成。

## 三层边界

1. **文件级**：私人 Markdown 使用 `sensitivity: private`。附件、目录或额外保险路径写入本地 `.releaseignore`，一行一个仓库相对路径。
2. **共享页面级**：索引、首页和地图中只属于私库的段落放在 `<!-- private:start -->` 与 `<!-- private:end -->` 之间；公开版需要的空状态写成单行 `<!-- release: 公开内容 -->`。
3. **反向检查级**：不得出现在公开副本中的名字、概念或标识写入本地 `.release-private-patterns`，构建时零命中才算通过。

`.releaseignore` 和 `.release-private-patterns` 可能直接暴露私人标题，已被 `.gitignore` 排除，发布包也不会包含它们。

## 构建公开原始版本

在仓库根目录运行：

```bash
python3 tools/build_public_release.py --version v1.0.0
```

脚本会：

- 排除 `.git`、发布产物、本地发布配置、明确排除路径和带 `sensitivity: private` 的非模板 Markdown；
- 移除共享页面的私密块，并启用公开空状态；
- 用空白[长期记忆模板](../模板/长期记忆.md)替换真实 `MEMORY.md`；
- 检查私密路径、敏感词、残留标记和 Markdown 内部链接；
- 在 `发布包` 中生成 ZIP 与 SHA-256 校验文件。

## 外发前必须确认

- ZIP 内没有 `.git`、`.releaseignore`、`.release-private-patterns` 或既有发布包。
- 真实 `MEMORY.md` 已被空白模板替代。
- 原始材料、思考卡、分析、复盘和共享导航中的私人痕迹都不在副本中。
- 内部链接检查通过，且敏感词清单零命中。
- 只发布新生成的 ZIP；不要把“从 Git 删除”误当作已经从历史中消失。

构建脚本是保护网，不代替人工判断。首次公开新主题、附件或大段引用前，仍要检查隐私、版权和引用范围。
