# 记录、拆分与关联工作流

本流程适用于“记一下、保存、整理、分析、写入仓库”等请求。它把已有的原始材料保护、卡片化和内部链接规则连接成一个默认动作：

> 先保存原始输入 → 识别内容形态 → 按独立意义拆分 → 搜索全库相关内容 → 建立有语义的链接 → 必要时更新长期记忆

## 1. 原始输入必须先有落点

只要后续笔记使用了用户提供的内容，就必须存在一个可回溯的原始材料记录。

- 短文字、单个命题或连续对话使用[轻量原始记录模板](../模板/轻量原始记录.md)。
- 长文、访谈、图片、PDF 或导出记录使用[原始材料模板](../模板/原始材料.md)。
- 同一连续主题中的多条用户消息可以按时间追加到同一份原始记录，不必每条消息新建文件。
- 一份原始材料可以被多张卡片共同引用，不复制原文。
- 默认只保存用户输入；模型回复只有在研究思想形成过程确有必要时才保存，并必须标明角色。
- 无法取得附件或完整上下文时保存可见部分，明确写出缺失，不用推测补全。
- 用户明确要求“只存原文”时，到原始材料和索引回链为止，不强制生成派生卡片。

原始材料可以低负担，但不能缺席。低负担意味着减少重复和字段，而不是用摘要替代原文。

## 2. 按内容形态决定是否拆分

拆分依据是独立复用价值，不是字数。

| 内容形态 | 主要去向 | 处理方式 |
| --- | --- | --- |
| 用户提出的独立判断、直觉或疑问 | `10-思考` | 一项中心命题一张思考卡片 |
| 概念、人物、作品、理论或案例知识 | `20-百科` | 建立或补充稳定实体条目 |
| 来自书籍、论文、网页或附件的材料 | `30-资料` | 保留来源笔记，不冒充个人观点 |
| 比较、综合论证、方案判断 | `40-分析` | 把多张卡片和来源组织成问题导向分析 |
| 面向读者的表达意图 | `50-写作` | 建立构思、草稿或项目总览 |
| 跨多条内容的导航关系 | `60-地图` | 更新已有地图；内容足够多时才新建地图 |

一段输入可以产生多种笔记。例如，一段读书感想可能同时形成一份原始记录、一份阅读笔记和两张思考卡片。不同文件各自承担一种职责，并共同链接到同一原始输入。

以下情况不拆：

- 两句话只有放在一起才构成一个完整判断；
- 拆开后每张卡片都无法脱离上下文理解；
- 只是为了让文件数量看起来更多；
- 用户明确要求只保存、不整理。

## 3. 卡片以用户观点为主体

思考卡片的内容顺序固定为：

1. **用户观点（主体）**：用户原话、用户自己的转述、理由、经验、保留意见和未决问题。
2. **AI 辅助内容（次要）**：澄清、概念区分、可能推论、反例、质疑、来源线索与写作建议。
3. **来源与仓库关联**：原始材料必需回链，其他链接写明关系。

约束：

- AI 不把用户的疑问改写成更强的肯定命题。
- 用户只确认核心命题时，不等于确认 AI 生成的全部论证。
- 未经确认的 AI 内容必须放在独立章节并标为“待确认”，不能混入“用户观点”。
- 当前没有足够用户观点时，卡片使用 `status: seed`，在主体区写“待用户补充或确认”；AI 内容不能反客为主。
- 用户否定某项 AI 展开时保留必要修订痕迹，不继续把它作为用户立场引用。

## 4. 每次拆分后搜索全库关联

新内容不是只放入一个文件夹；完成派生笔记前，必须进行一次仓库内搜索。

### 搜索顺序

1. **直接匹配**：标题、别名、用户原句、关键短语。
2. **概念匹配**：同义词、上位概念、相反概念、关键人物与作品。
3. **问题匹配**：是否回答同一个问题，是否依赖相同前提或面对相同反例。
4. **来源匹配**：是否来自同一材料，或引用同一作者、作品和事件。
5. **导航匹配**：相关主题地图、首页当前焦点和 [`MEMORY.md`](../MEMORY.md) 中的长期线索。

搜索范围包括实际知识笔记与归档的文件名、YAML `title` / `aliases` / `tags` / `related`、正文、索引、地图和长期记忆。归档参与查重；模板、规则、构建工具和发布包默认排除，避免操作说明制造伪关联。私密原始材料只在必要范围内参与检索，不把敏感文本复制到公开笔记。

### 关系类型

链接旁至少写明一种关系：

- `来源于 / 派生为`：原始材料与派生笔记之间的强制双向关系
- `同一命题 / 修订自`：同一判断的重述、演化或版本关系
- `支持`：为本命题提供理由或证据
- `扩展`：把本命题推进到新的层次或场景
- `修正`：缩小范围、增加条件或改变原结论
- `冲突`：给出不相容的判断或证据
- `前提`：本笔记依赖它才能成立
- `反例`：挑战本命题的普遍性
- `例证`：提供作品、事件或工程案例
- `应用`：把思想用于分析、决策或写作
- `同源`：来自同一份原始材料或来源
- `类比`：共享结构但对象不同；必须同时写明类比边界
- `同一问题`：结论不同，但试图回答同一个问题

只有能写出一句具体关系说明时才建立链接。共同出现一个关键词不等于有关联。

### 没有找到关联时

不要强行链接。在笔记的“仓库关联”中记录：检索日期、检索范围、关键词或同义词、`暂未找到直接相关内容`、孤立原因（新主题／资料不足／表述独特等）和下次重查条件。可以链接领域索引作为导航，但必须说明“导航入口，不是语义关联”。没有关联是一项搜索结果，不是失败。

## 5. 建立回链和导航

- 每张派生卡片必须链接原始材料。
- 原始材料的“派生笔记”必须回链到卡片、分析或写作。
- 与已有笔记的关系在新笔记正文中写清；关系重要时再给旧笔记补反向入口。
- 新内容进入对应索引；能改变一个主题的理解或导航时才进入主题地图。
- 不为满足“双向链接”而机械修改所有旧文件。

## 6. 判断是否更新长期记忆

[`MEMORY.md`](../MEMORY.md) 只记录能跨多次任务持续有用的学术方向、用户已确认的核心命题、研究进展和长期未决问题。

满足以下任一条件才考虑更新：

- 用户明确确认了一个会反复使用的核心观点；
- 一个研究方向出现实质推进、转折或放弃；
- 新笔记改变了多个既有主题之间的关系；
- 一个长期问题进入或离开当前焦点；
- 用户明确要求更新长期记忆。

一次性的闪念、未确认的 AI 推演、完整原文和细碎进度不写入长期记忆。`MEMORY.md` 是轨迹索引，不是第二份正文或证据库。

## 7. 处理完成后的反馈

每次记录任务完成后，助手简要告诉用户：

- 原始材料保存在哪里；
- 拆成了哪些卡片或其他笔记；
- 搜索到了哪些实质关系，以及关系类型；
- 哪些内容是用户观点，哪些仍是 AI 辅助、待确认；
- 是否更新了 `MEMORY.md`，若没有则说明未达到长期记忆门槛。
