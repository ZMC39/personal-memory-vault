# 思考和整理

这是一个用来保存、连接和发展个人思想的长期记忆库。它面向哲学、软件架构、文学等主题，也允许未来自然扩展到新的领域。

这里不追求一开始就把所有内容分得完美，而是让内容沿着一条稳定路径逐渐成熟：

> 捕捉 → 澄清 → 连接 → 沉淀 → 分析或写作 → 归档

## 从这里开始

1. 不清楚怎样使用时，直接打开[记忆库助手使用指南](助手使用指南.md)，照自然语言示例提问。
2. 打开[首页](首页.md)，从当前问题、长期记忆或主题地图进入。
3. 来不及整理的内容先放进 [00-收件箱](00-收件箱/README.md)。只要内容要形成正式笔记，先保存一份可回溯的原始输入。
4. 记录后默认按[记录、拆分与关联工作流](规则/记录与关联工作流.md)处理：识别内容形态、按独立意义拆分、逐条搜索全库并建立语义链接。
5. 有一个相对完整的个人观点时，使用[思考卡片模板](模板/思考卡片.md)；用户观点放在主体，AI 澄清与推演放在独立辅助区。
6. 整理稳定知识时，使用[百科条目模板](模板/百科条目.md)并遵循[百科编写规则](规则/百科编写规则.md)。
7. 开始专题研究或长篇表达时，分别进入 [40-分析](40-分析/_索引.md) 或 [50-写作](50-写作/_索引.md)。
8. 当同一种整理摩擦反复出现时，先记录并按[知识库演化规则](规则/知识库演化规则.md)进行最小试验。
9. 需要公开空白版本时，按[发布与隐私规则](规则/发布与隐私规则.md)从私库生成净化副本。

## 目录

| 目录 | 用途 |
| --- | --- |
| `00-收件箱` | 尚未分类的闪念、问题、摘录和待处理材料 |
| `10-思考` | 以“自己的判断”为核心的原子化思考卡片 |
| `20-百科` | 相对客观、可复用、带来源的概念、人物、作品与理论条目 |
| `30-资料` | 原始材料、阅读笔记、引文及外部资料的入口 |
| `40-分析` | 围绕问题进行比较、论证、综合和决策 |
| `50-写作` | 从构思、草稿、项目到完成稿的写作空间 |
| `60-地图` | 主题地图与跨笔记导航，负责把知识连接起来 |
| `80-复盘` | 周期性回顾、未决问题和知识库演化记录 |
| `90-归档` | 已失效、已合并或暂时不再活跃的内容 |
| `模板` | 各类笔记的可复制骨架 |
| `规则` | 分类、命名、元数据、百科与引用规则 |
| `MEMORY.md` | 私密的长期学术脉络、已确认命题、研究进展与长期问题 |
| `助手使用指南.md` | 不记目录也能直接使用的自然语言操作入口 |
| `发布包` | 去隐私公开副本及其校验文件的本地输出位置 |

## 核心原则

- **先记录，后分类。** 分类不确定时一律进入收件箱，避免因为整理成本丢掉想法。
- **原始输入先行。** 正式派生内容必须能回到原文或原件记录；低负担保存不等于只留摘要。
- **一条笔记解决一个中心问题。** 这样内容才能被重复引用、组合和修订。
- **用户观点为主，AI 辅助为辅。** 用户原话、已确认观点、AI 推演与待确认内容分区保存。
- **关联必须说明关系。** 新卡片逐条搜索仓库；找不到实质关联时如实记录，不制造链接。
- **类别可增也可减。** 新增、拆分、合并或退役类别都先盘点真实内容；减少类别只改变活动导航，材料迁移到新位置或归档。
- **用户材料默认不可删除。** 除非用户明确指定永久删除对象，整理、去重、重构、回滚和发布都只能移动、重命名或归档。
- **事实、他人观点和个人判断分开写。** 记忆库不仅保存结论，也保留结论的来源和形成过程。
- **规则也可以被修订。** 只依据真实摩擦，通过最小、可验证、可回滚的试验演化。

完整约定见[知识库规则](规则/知识库规则.md)和[记录、拆分与关联工作流](规则/记录与关联工作流.md)。
